# AICoding 实施任务拆分模板

在当前 SDD 工作流指定的目标成果物 TOBE 章节中使用本模板；如果工作流未指定，默认用于 `xxx模块详细设计说明书`。任务必须能被后续 AICoding 直接执行。

## 任务拆分规则

- 每个任务只承载一个清晰目标。
- 任务之间要标明依赖关系。
- 任务必须落到具体文件、包、类、函数或明确的新建位置。
- 任务必须包含验收标准和建议测试。
- 任务必须能关联到测试矩阵中的至少一个测试项，除非明确说明该任务无需测试的理由。
- 每个任务必须包含阶段性验证点，优先设计能在小步实现后立即运行的单元测试、契约测试或局部集成测试。
- 高风险任务必须说明风险来源和验证方式。
- 不要使用“优化”“完善”“处理一下”这类不可执行表述。

## 验收标准写法

验收标准应描述可验证行为，例如：

- 当输入满足某条件时，目标接口返回指定状态或字段。
- 当外部依赖失败时，系统按指定错误码、重试或降级策略处理。
- 当配置开关关闭时，保持 ASIS 行为不变。
- 当迁移完成后，新旧数据读写兼容。
- 指定单元测试、集成测试或契约测试通过。
