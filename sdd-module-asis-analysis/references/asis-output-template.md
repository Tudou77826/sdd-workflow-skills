# 模块详细设计说明书 ASIS 双文件模板

更新当前 SDD 工作流指定的模块详细设计双文件时使用本模板；如果工作流未指定，默认用于 `{AR编号}-{需求短名}-{模块名}模块详细设计说明书.md` 和同名前缀 `.context.md`。

正式说明书是唯一开发依据。正式说明书中的 ASIS 只保留会影响 TOBE、编码任务、验证或风险控制的关键结论；`.context.md` 记录详细证据、检索过程、边界确认记录、调用链展开和完整证据索引。禁止将影响开发的 ASIS 结论只写入 `.context.md`。

小范围变更或小模块模式可以合并章节，但正式说明书至少保留：分析结论摘要、需求/AR 与 ASIS 映射摘要、模块边界与本次分析切片、关键 ASIS 事实、隐藏约束与风险、测试覆盖现状、待确认问题。不要为了套完整模板而生成大量空表。
