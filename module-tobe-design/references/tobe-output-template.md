# 模块详细设计说明书 TOBE 正式模板

使用本模板更新当前 SDD 工作流指定的正式《{AR编号}-{需求短名}-{模块名}模块详细设计说明书.md》。正式说明书是指导开发的最后一道文档，也是后续 AICoding 的唯一开发依据；只有 TOBE 阶段可以编辑正式说明书正文。ASIS 和 Gate 阶段只写同名前缀 `.context.md` 或返回检查结论。

TOBE 必须基于 `.context.md` 中的 ASIS 证据形成设计，不得脱离证据凭空设计。正式说明书中可以引用 ASIS 结论和证据编号，但不得展开 ASIS 检索过程。`.context.md` 只记录证据、推导和过程，不参与后续开发指导。

一级标题遵循本模板。未涉及的章节保留标题并写明“不涉及，原因：...”，不要生成大量空表。第 8 章是 AICoding 的核心开发依据，必须按需求点展开，不能只用一个总表替代正文。
